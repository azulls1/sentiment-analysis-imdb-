# PRD Checklist

Use esta checklist para validar que el PRD está completo y es de calidad.

## Estructura

- [ ] Tiene portada con metadata (versión, fecha, autor, estado)
- [ ] Tiene tabla de contenidos
- [ ] Secciones numeradas correctamente
- [ ] Formato consistente en todo el documento
- [ ] Usa terminología consistente

## Contenido Obligatorio

### Visión y Objetivos
- [ ] Visión del producto clara y concisa
- [ ] Objetivos de negocio definidos
- [ ] KPIs medibles para cada objetivo
- [ ] Criterios de éxito claramente establecidos

### Usuarios
- [ ] Al menos 1 user persona detallada
- [ ] Cada persona tiene: rol, necesidades, pain points
- [ ] User journey documentado (si aplica)

### Requisitos Funcionales
- [ ] Todos los requisitos tienen ID único (RF-XXX)
- [ ] Cada requisito tiene descripción clara
- [ ] Priorización MoSCoW aplicada a todos
- [ ] Requisitos agrupados por epic/feature
- [ ] Criterios de aceptación para requisitos clave

### Requisitos No Funcionales
- [ ] Performance definido con métricas
- [ ] Seguridad especificada
- [ ] Escalabilidad considerada
- [ ] Disponibilidad/SLAs definidos
- [ ] Usabilidad/Accesibilidad considerada

### Alcance
- [ ] Out of scope claramente listado
- [ ] Límites del MVP definidos
- [ ] Futuras versiones mencionadas (si aplica)

### Dependencias y Riesgos
- [ ] Dependencias externas identificadas
- [ ] Dependencias técnicas documentadas
- [ ] Riesgos identificados
- [ ] Plan de mitigación para cada riesgo

### Timeline
- [ ] Fases definidas
- [ ] Milestones establecidos
- [ ] Entregables por fase

## Calidad del Contenido

- [ ] Sin ambigüedades (cada requisito es interpretable de una sola forma)
- [ ] Sin duplicados (requisitos no se repiten)
- [ ] Trazabilidad (cada requisito tiene origen justificado)
- [ ] Testeable (cada requisito puede verificarse)
- [ ] Completo (cubre todos los aspectos del producto)

## Revisión Final

- [ ] Spell check realizado
- [ ] Formato consistente
- [ ] Links funcionan
- [ ] Diagramas son legibles
- [ ] Revisado por stakeholders (si aplica)

---

## Notas de Uso

- Usar esta checklist ANTES de dar por terminado el PRD
- Marcar items como N/A si no aplican (con justificación)
- Un PRD sin todos los items marcados NO está listo para desarrollo
